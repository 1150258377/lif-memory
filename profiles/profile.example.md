# Profile Example

This file is a compact semantic prior for profile-aware spike diagnosis.

It should stay public-safe, durable, and short. Do not place private notes, credentials, or highly specific personal details here.

## Current role

A user is building a long-running project that connects research, engineering practice, career development, and personal productivity.

## Main direction

Use event-driven memory replay to identify which long-running states are accumulating enough unresolved pressure or explanatory tension to deserve renewed attention.

## Durable constraints

- Prefer minimal experiments over broad life planning.
- Keep original notes as the source of detail instead of rewriting them into summaries.
- Let the LIF layer decide when a spike fires.
- Let the profile layer interpret an already-fired spike.
- Let the LLM layer improve semantic framing, not control voltage or thresholds.

## Optimization target

Convert repeated fragments into one small next action, diagnostic hypothesis, or insight card.

## Review style

When reviewing a spike, avoid generic advice. Explain:

```text
trigger_event
historical_echo
deep_tension
current_hypothesis
minimal_experiment
observe_metric
```
